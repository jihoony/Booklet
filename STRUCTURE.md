# 코드 구조 및 기능 매핑

이 문서는 `main.go`의 소스코드 섹션이 `Booklet Creator`의 어떤 기능에 해당하는지 설명합니다.

## 파일 구성

- `main.go`: CLI 핸들링, PDF 프로세싱, 가이드라인 생성 로직을 모두 포함하는 메인 파일입니다.
- `go.mod`: 프로젝트 의존성 관리 파일입니다.
- `README.md`: 기본 사용법 및 설치 안내서입니다.

## 기능별 코드 매핑

| 기능 | 코드 섹션 / 함수 | 설명 |
| :--- | :--- | :--- |
| **CLI 인자 파싱** | `main()` (17-70행) | `-i`, `-o`, `-n`, `-formsize`, `-btype` 등의 플래그 정의 및 처리. |
| **유효성 검사** | `main()` (72-75행) | `-n` 값이 유효한지(2, 4, 6, 8) 검사. |
| **자동 Multifolio** | `main()` (85-106행) | 전체 페이지 수를 계산하여 시그니처 모드 자동 활성화 여부 결정. |
| **소책자 엔진 호출** | `main()` (132-143행) | `pdfcpu` 라이브러리를 사용하여 소책자용 페이지 재배치 수행. |
| **가이드라인 적용** | `addGuides()` | 생성된 가이드라인을 결과 PDF 위에 워터마크로 입히는 고수준 함수. |
| **PDF 생성기** | `createGuidePDF()` | 선을 그리기 위한 저수준 PDF 바이트 스트림 생성. |
| **좌표 계산 로직** | `createGuidePDF()` switch문 | 각 n-up 레이아웃별 실선/점선 위치 기하학 정의. |

## 외부 라이브러리 활용

이 프로젝트는 `github.com/pdfcpu/pdfcpu`에 크게 의존하고 있습니다. 주요 활용 API는 다음과 같습니다.
- `api.PDFBookletConfig`: 소책자 구성 객체 생성.
- `api.BookletFile`: 실제 페이지 재배치 및 병합 수행.
- `api.AddWatermarksFile`: 페이지 상단에 가이드라인 "스탬프" 추가.
- `api.ReadContextFile`: PDF의 페이지 수, 크기(MediaBox) 등 메타데이터 분석.
