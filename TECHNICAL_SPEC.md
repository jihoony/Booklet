# 기술 사양 (Technical Specification)

## 1. 개요 (Overview)
`Booklet Pro`는 기존 CLI 기반의 소책자 제조 로직을 현대적인 데스크탑 애플리케이션 환경으로 확장한 프로젝트입니다. 핵심 엔진은 Go로 구현되어 성능과 안정성을 보장하며, 사용자 인터페이스는 웹 기술을 활용하여 직관적이고 풍부한 경험을 제공합니다.

## 2. 기술 스택 (Technology Stack)
### Backend
- **언어**: Go (1.25+)
- **데스크탑 프레임워크**: [Wails v2](https://wails.io/)
- **PDF 엔진**: [pdfcpu](https://github.com/pdfcpu/pdfcpu)

### Frontend
- **프레임워크**: Vanilla JS/TS (Vite 환경)
- **스타일링**: CSS3 (Modern Glassmorphism Design)
- **빌드 도구**: Vite

## 3. 시스템 아키텍처 (Architecture)
프로젝트는 크게 세 가지 계층으로 분리되어 있습니다:

### 1) 핵심 비즈니스 로직 (`pkg/booklet`)
- PDF 페이지 재배치(Reordering) 알고리즘
- N-Up 레이아웃 계산 로직
- 시각적 가이드라인 생성 및 워터마킹 엔진

### 2) 앱 브릿지 계층 (`app.go`)
- 프런트엔드에서 호출 가능한 Go 메서드 정의
- OS 표준 파일 다이얼로그(Open/Save) 연동
- 시스템 폴더 열기 등 네이티브 기능 제어

### 3) 프런트엔드 인터페이스 (`frontend/`)
- 상태 관리 및 사용자 설정 수집
- Wails 런타임을 통한 백엔드 API 호출
- 반응형 레이아웃 및 커스텀 모달 알림창

## 4. 핵심 알고리즘 및 로직
### 소책자 변환 로직
- 입력 PDF의 총 페이지 수를 기반으로 4의 배수(또는 N의 배수)로 패딩을 계산합니다.
- 중첩된 폴리오(Folio) 구조를 생성하여 앞뒷면 인쇄 시 페이지가 올바른 순서로 오도록 재배치합니다.

### 가이드라인 생성
- 각 면의 중앙(접는 선)과 가장자리(절단선) 좌표를 수학적으로 계산하여 투명 PDF 레이어로 생성 후 원본에 오버레이합니다.

## 5. 실행 환경 감지
`main.go`는 실행 시 전달된 명령줄 인자를 분석합니다.
- 인자가 존재하는 경우: `runCLI()`를 호출하여 터미널 모드로 동작합니다.
- 인자가 없는 경우: `runGUI()`를 호출하여 데스크탑 앱을 띄웁니다.
