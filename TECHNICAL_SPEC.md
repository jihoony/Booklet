# 기술 사양서: Booklet Creator

이 문서는 Booklet Creator CLI 도구의 내부 동작 원리와 구현 세부 사항을 설명합니다.

## 아키텍처 개요

이 도구는 Go 언어로 작성되었으며, PDF 조작을 위해 주로 `pdfcpu` 라이브러리에 의존합니다. 전체적인 실행 흐름은 다음과 같습니다.

1.  **인자 파싱**: `flag` 패키지를 사용하여 입력/출력 경로, 레이아웃 옵션(`-n`), 용지 크기, 제본 방식 및 유형(`-btype`) 등을 처리합니다.
2.  **설정 준비**: 사용자 입력을 바탕으로 `pdfcpu` 전용 설정 문자열을 생성합니다.
3.  **자동 시그니처 감지**: 입력 PDF의 페이지 수를 분석하여 `multifolio`(여러 묶음으로 나누기) 모드 활성화 여부를 결정합니다.
4.  **핵심 변환**: `pdfcpu` API를 호출하여 페이지 순서를 재배치하고 지정된 n-up 레이아웃으로 병합합니다.
5.  **가이드라인 후처리**: (선택 사항) 절단선과 접는 선이 포함된 오버레이 PDF를 생성하여 결과물에 스탬프(Stamp) 방식으로 추가합니다.

## 주요 구성 요소

### 1. 메인 실행 로직 (`main.go`)

`main` 함수는 변환의 전체 라이프사이클을 관리합니다.

*   **Multifolio 로직**:
    ```go
    pagesPerSheet := n * 2
    totalSheets := (pageCount + pagesPerSheet - 1) / pagesPerSheet
    if totalSheets > 10 {
        multifolio = "on"
    }
    ```
    이 로직은 두꺼운 문서가 한 번에 접기 힘든 점을 고려하여, 자동으로 관리 가능한 단위(기본 6장)로 나누어 출력되도록 합니다.

*   **PDF 생성**:
    `api.PDFBookletConfig`를 사용하여 설정을 생성하고, `api.BookletFile`을 호출하여 실제 PDF 변환을 수행합니다.

### 2. 가이드라인 생성 (`addGuides` 및 `createGuidePDF`)

`pdfcpu`의 기본 가이드라인 대신, 텍스트가 없는 깔끔한 디자인을 위해 커스텀 가이드라인을 구현했습니다.

*   **저수준 PDF 구성**: `createGuidePDF` 함수는 바이트 단위로 PDF 파일을 직접 빌드합니다.
    - **Catalog & Pages**: PDF의 기본 구조 정의.
    - **Content Stream**: 그리기 명령(예: 이동 `m`, 선 긋기 `l`, 스트로크 `S`) 정의.
    - **스타일**: 자르는 선은 실선(`[] 0 d`), 접는 선은 점선(`[3] 0 d`)으로 설정.

*   **좌표 계산 로직**:
    `n`(한 면당 페이지 수)과 `binding`(제본 방향)에 따라 선의 위치가 달라집니다.
    - `n=2`: 중앙에 가로 점선(접는 선).
    - `n=4` (Long Edge): 가로 실선(자르는 선), 세로 점선(접는 선).
    - `n=8`: 1/2, 1/4, 3/4 지점에 복합적인 선 배치.

### 3. 워터마킹(스탬핑)

생성된 가이드라인 PDF는 `api.AddWatermarksFile`을 통해 결과 PDF에 적용됩니다. 이를 통해 원본 내용의 레이아웃 로직을 건드리지 않고 모든 페이지 위에 선을 정확히 겹쳐 그릴 수 있습니다.

## 의존성

- **github.com/pdfcpu/pdfcpu**: PDF 파싱, 페이지 재배치, 워터마킹을 담당하는 핵심 엔진.
- **Go 표준 라이브러리**: CLI 플래그, I/O 처리, 수동 PDF 바이트 구성 등에 사용.

## 설계 결정 사항

- **가이드라인 직접 생성 이유**: 외부 PDF 생성 라이브러리(예: gofpdf)에 대한 의존성을 줄이고 바이너리 크기를 최적화하기 위함입니다. 또한 PDF 콘텐츠 스트림을 직접 제어하여 파일 용량 증가를 최소화했습니다.
- **자동 Multifolio**: 실제 소책자를 만들 때 10~12장 이상의 종이를 한꺼번에 접으면 종이가 밀려나는 현상이 발생하므로, 물리적인 제본 품질을 위해 필수적인 기능입니다.
